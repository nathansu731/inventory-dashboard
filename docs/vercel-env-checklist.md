# Vercel Env Checklist

Use this checklist when setting up the dashboard on Vercel for `develop` and `production`.

## Backend Generated

These should come from backend Terraform output export, not be typed by hand:

- `AWS_REGION`
- `COGNITO_REGION`
- `COGNITO_USER_POOL_ID`
- `APPSYNC_API_URL`
- `NEXT_PUBLIC_GRAPHQL_ENDPOINT`
- `S3_RAW_BUCKET`
- `TENANTS_TABLE`
- `NOTIFICATIONS_TABLE`
- `SAVED_REPORTS_TABLE`
- `DATA_SOURCES_TABLE`
- `ENTITLEMENTS_TABLE`
- `WORKER_CRON_TOKEN`

Notes:

- `NEXT_PUBLIC_GRAPHQL_ENDPOINT` should equal `APPSYNC_API_URL`.
- `WORKER_CRON_TOKEN` is generated from backend config, but it is still a secret.

## Frontend Only Plain Text

These are not generated by backend Terraform in this repo. Set them manually per environment:

- `COGNITO_CLIENT_ID`
- `NEXT_PUBLIC_COGNITO_CLIENT_ID`
- `NEXT_PUBLIC_COGNITO_AUTHORITY`
- `NEXT_PUBLIC_COGNITO_DOMAIN`
- `NEXT_PUBLIC_COGNITO_REDIRECT_URI`
- `NEXT_PUBLIC_COGNITO_LOGOUT_URI`
- `NEXT_PUBLIC_COGNITO_SCOPE`
- `STRIPE_PRICE_ID_LAUNCH`
- `STRIPE_PRICE_ID_PROFESSIONAL`
- `STRIPE_PRICE_ID_ENTERPRISE`
- `NEXT_PUBLIC_STRIPE_PRICE_ID_LAUNCH`
- `NEXT_PUBLIC_STRIPE_PRICE_ID_PROFESSIONAL`
- `NEXT_PUBLIC_STRIPE_PRICE_ID_ENTERPRISE`
- `AWS_SES_FROM_EMAIL`
- `HELP_CENTER_FORWARD_TO_EMAIL`
- `SHOPIFY_SCOPES`
- `SHOPIFY_REDIRECT_URI`
- `QUICKBOOKS_SCOPES`
- `QUICKBOOKS_REDIRECT_URI`
- `BIGCOMMERCE_SCOPES`
- `BIGCOMMERCE_REDIRECT_URI`
- `AMAZON_REDIRECT_URI`
- `AMAZON_SELLER_CENTRAL_URL`
- optional `AMAZON_SP_API_BASE_URL`
- optional `AMAZON_SP_API_REGION`

Notes:

- `COGNITO_CLIENT_ID` and `NEXT_PUBLIC_COGNITO_CLIENT_ID` should match.
- `STRIPE_PRICE_ID_*` and `NEXT_PUBLIC_STRIPE_PRICE_ID_*` should match by plan.

## Secrets

These should be added to Vercel as secrets for each environment:

- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- optional `AWS_SESSION_TOKEN`
- `COGNITO_CLIENT_SECRET`
- `DATA_SOURCE_ENCRYPTION_KEY`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`
- `SHOPIFY_CLIENT_ID`
- `SHOPIFY_CLIENT_SECRET`
- `QUICKBOOKS_CLIENT_ID`
- `QUICKBOOKS_CLIENT_SECRET`
- `BIGCOMMERCE_CLIENT_ID`
- `BIGCOMMERCE_CLIENT_SECRET`
- `AMAZON_SP_APPLICATION_ID`
- `AMAZON_LWA_CLIENT_ID`
- `AMAZON_LWA_CLIENT_SECRET`
- `AMAZON_SP_AWS_ACCESS_KEY_ID`
- `AMAZON_SP_AWS_SECRET_ACCESS_KEY`
- optional `AMAZON_SP_AWS_SESSION_TOKEN`

## Validation Notes

- `DATA_SOURCE_ENCRYPTION_KEY` must decode to exactly 32 bytes.
- `WORKER_CRON_TOKEN` must match the EventBridge API destination auth if the scheduled sync worker is enabled.
- Amazon SP-API should use dedicated `AMAZON_SP_AWS_*` credentials in prod, not generic app AWS credentials.
- Vercel still needs AWS credentials unless you move to another AWS auth mechanism.

## Vercel Dev Setup

1. Select the dev Terraform workspace.
2. Run:
   `node ../forecasting-core/ops/export-dashboard-env.js --env develop`
3. Copy values from `.env.develop.generated` into Vercel `Development`.
4. Add dev-only manual plain-text values.
5. Add dev-only secrets.
6. Run:
   `npm run validate:develop`

Recommended Vercel target:

- `Development` for local/dev dashboard
- optionally `Preview` if your preview deploys should talk to dev backend

## Vercel Prod Setup

1. Select the prod Terraform workspace.
2. Run:
   `node ../forecasting-core/ops/export-dashboard-env.js --env production`
3. Copy values from `.env.production.generated` into Vercel `Production`.
4. Add prod-only manual plain-text values.
5. Add prod-only secrets.
6. Run:
   `npm run validate:production`

## Optional Preview Strategy

Choose one and keep it consistent:

- Preview uses dev backend envs.
- Preview uses its own isolated backend envs.

Do not point preview at prod accidentally.

## Suggested Environment Mapping

- Vercel `Development` -> dev backend + dev Cognito + dev Stripe + dev connectors
- Vercel `Preview` -> usually dev backend + dev/shared non-prod integrations
- Vercel `Production` -> prod backend + prod Cognito + prod Stripe + prod connectors

## Post-Setup Checks

After envs are added:

1. Run:
   `npm run validate:all-envs`
2. Deploy the dashboard.
3. Run smoke flow against the deployed environment:
   `DASHBOARD_BASE_URL=... DASHBOARD_COOKIE_HEADER='id_token=...; access_token=...; refresh_token=...' npm run smoke:dashboard-flow`
